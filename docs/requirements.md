# 产品需求（Requirements）

## 背景

为浙江大学 ZIBS（浙江大学国际联合商学院）稳定币研究中心打造的双语（中/英）学术研究平台。面向学术访问者，用于发布研究成果、策展全球稳定币相关文献，并展示监管与市场数据。

## 目标用户

- 学术访问者 / 研究人员（主要消费内容，无需登录即可浏览已审核通过的内容）
- 注册用户（可提交文献、查看自己提交的待审核内容）
- 管理员（审核文献、管理用户角色、发布 ZIBS 自有研究）

## 核心能力

1. **ZIBS 自有研究发布**（`our_research` 表）—— 管理员上传/管理本中心的研究成果。
2. **全球文献库**（`resources` 表）—— 用户提交文献条目，管理员审核（pending/approved/rejected）后对外展示；支持 AI 辅助导入（粘贴 URL，调用 Gemini 自动抽取标题/作者/摘要/标签）。
3. **作者/学者档案**（`authors` + `institutions` 表）—— 文献作者自动归集为学者档案，关联所属机构，可在 `/experts` 浏览、在 `/author/:id` 查看个人主页。
4. **监管与市场数据展示**（`/regulatory`、`/market-data`、`/quantitative`、`/dashboard` 页面）。
5. **账号体系**——注册/登录/找回密码，区分 `user` / `admin` 角色。
6. **双语界面**——所有用户可见文案需同时提供中英文。

## 内容审核工作流（重要业务逻辑）

- 非管理员创建的 `resources` 条目默认状态为 `pending`，管理员创建的默认为 `approved`。
- 未登录用户只能看到 `approved` 的条目；登录的普通用户能看到 `approved` 的条目 + 自己创建的条目（任意状态）；管理员能看到全部，并可按 `status` 过滤。
- 非管理员编辑已有条目后，状态会被重置为 `pending`，需要重新审核。
- 管理员通过 `PATCH /api/resources/:id/approve` 来批准/拒绝。

这一审核流程是业务核心约束，不是实现细节——改动前需确认是否会破坏审核语义，详见 [`api-design.md`](./api-design.md) 和 [`database.md`](./database.md)。
