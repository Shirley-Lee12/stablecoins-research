# Code 说明 ── 上传第四入口:题录文件导入(知网 RIS/EndNote/NoteExpress)

> 给中文文献(尤其知网)用的导入入口。复用已有管线的后半段(打标 → 完整性核对 → upload_jobs → 确认入库),只新增"题录解析"前处理。
> 核心:数据源是知网,元数据本身权威 → **跳过联网反查/真实性核对**,只做完整性核对。

---

## 一、sourceType 枚举扩充(先做)

`resources` 的 `sourceType` 增加两个值:
- `会议论文 Conference Paper`
- `学位论文 Thesis`

最终全集:`期刊论文 / 工作论文 / 会议论文 / 学位论文 / 报告 / 政府文件 / 新闻 / 智库报告`

> 概念澄清(供归类时参考):会议论文(发表场合)与工作论文(发表阶段)是平行维度,不互相包含。进正式会议论文集 → 会议论文;仅机构挂出的预印本/讨论稿 → 工作论文;拿不准归"工作论文"兜底,或在确认弹窗人工改。报纸归"新闻"。

题录类型映射(`RT`/`%0`/`{Reference Type}` 的值 → sourceType):
| 题录类型 | sourceType |
|---|---|
| Journal Article | 期刊论文 |
| Conference Proceeding(s) | 会议论文 |
| Dissertation/Thesis / Thesis | 学位论文 |
| Newspaper Article | 新闻 |
| Report | 报告 |
| Book | (新增 Book 或归报告,按需) |

## 二、统一题录解析器(`lib/citation/`)

四种知网导出格式本质是"标签:值"纯文本,只有三套语法(RefWorks 与"知网研学"标签完全相同)。**自动嗅探格式,不要求用户指定。**

### 嗅探规则(看首个非空记录的起始标签)
- 以 `RT ` 开头 → RefWorks / 知网研学(同一套解析)
- 以 `%0` 开头 → EndNote
- 以 `{Reference Type}` 开头 → NoteExpress

### 三套字段映射表 → 标准六要素
| 标准字段 | RefWorks/知网研学 | EndNote | NoteExpress |
|---|---|---|---|
| 类型 | `RT` | `%0` | `{Reference Type}` |
| 标题 | `T1` | `%T` | `{Title}` |
| 作者 | `A1` | `%A`(多行) | `{Author}` |
| 机构 | `AD` | `%+` | `{Author Address}` |
| 期刊/来源 | `JF` | `%J` | `{Journal}` / `{Secondary Title}` |
| 年 | `YR` | `%D` | `{Year}` |
| 期 | `IS` | `%N` | `{Issue}` |
| 页 | `OP` | `%P` | `{Pages}` |
| 关键词 | `K1` | `%K` | `{Keywords}` |
| 摘要 | `AB` | `%X` | `{Abstract}` |
| DOI | `DO` | `%R` | `{DOI}` |
| 链接 | `LK` | `%U` | `{URL}` |
| 报纸日期 | `FD` | `%8` | `{Date}` |
| 出版方 | `PB` | `%I` | `{Publisher}` |
| 语言 | `LA` | — | — |

### 解析细节(这批真实样本暴露的坑,必须处理)
1. **记录分隔**:空行分隔多条;每条以类型标签起始
2. **编码兜底**:知网导出常为 GBK 而非 UTF-8 → 先按 UTF-8 解,乱码则按 GBK 重解
3. **多作者拆分**:
   - RefWorks/NoteExpress:`A1` 值按 `;` 分号拆(注意 NoteExpress 作者间有大量缩进空格,需 trim)
   - EndNote:多行 `%A` 各为一位作者
   - 例:`谭学想;曹丽红` → ["谭学想","曹丽红"];李施爱那条 3 位
4. **机构作者**:无作者标签(如"行业热搜"那条)→ 用 `PB` 或 `JF` 作为 organization 型作者兜底(对接已认可的 authors.type=organization;当前先存 authorRaw 临时字段)
5. **缺字段容忍**:报纸/会议论文常缺 `AB`、`IS`、`OP` → 不报错,留空进后续补全逻辑

## 三、导入流程(接 upload_jobs,跳过联网验证)

题录文件批量导入,走 upload_jobs(可能几十上百条):
1. 用户上传 `.ris/.txt/.enw` 文件 → 解析器拆成 N 条记录
2. 每条建 upload_jobs 行,解析出的六要素写 extractedData
3. **跳过 U.2 反查链接、跳过 U.4 的联网真实性核对**(DOI/链接来自知网,直接当直达标识存)
4. 走打标(retagResources 的 computeTagsForText:中文摘要+中文关键词 → 中文标签定义句相似度匹配;币种/辖区受控匹配)
5. 走**完整性核对**(只查六要素齐不齐,不联网):
   - 硬必填(标题+≥1作者/机构+年份)缺 → failed
   - 摘要缺 → 进"摘要补全"(见下),仍可入库标 needs_review
6. 确认弹窗逐条展示 → 用户确认 → 批量入 resources

## 四、缺摘要的两层补全(报纸专用,守住边界)

报纸类常无 `AB`,且多为公开新闻、网上能搜到同一篇全文 → 可提炼。但守住边界防 AI 编造:

**第一层:先用题录自带文本。** 若有 `K1`(关键词),打标直接用关键词+标题,**不必联网**。很多场景到这层就够了。

**第二层:联网补摘要,但只在"抓到全文"时提炼。**
- 用标题(+作者/报社)联网检索,尝试抓取**同一篇全文**
- **抓到全文** → LLM 基于全文做提炼性摘要,写入 abstract,**标注 `abstractSource: "generated_from_fulltext"`**
- **只搜到标题/片段、抓不到全文** → **留空**,标 needs_review,**绝不凭标题生成摘要**
- 抓取到的若是登录墙/相似但非同一篇 → 视为未抓到,留空

> 边界一句话:**提炼基于真实全文,不基于标题猜测。** 抓不到全文宁可空着等人工。`abstractSource` 字段区分 `cnki`(题录自带)/`generated_from_fulltext`(工具提炼)/`manual`(人工补),便于追溯。

## 六、第三批测试夹具(`test-fixtures-batch3-cnki.json`)

- **解析正确**:四格式各取若干,断言六要素拆分正确(尤其多作者:谭学想 2 人 / 李施爱 3 人;机构作者:"行业热搜")
