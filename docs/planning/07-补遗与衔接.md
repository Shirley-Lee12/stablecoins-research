# 一、authorRaw 临时字段(四个入口统一,现在就加)

**背景**:政府文件/法规/机构报告没有个人作者(发布主体是机构,如 European Parliament、HKMA),会撞上"≥1 作者"硬必填。正式的 authors 实体表排在上传管线之后才建(见第二节),所以现在先用临时字段兜着,不阻塞入库。

**做法**:
- `resources` 加字段 `authorRaw`(建议 `jsonb`,存 `[{name, type}]`;`type` ∈ `person|organization`,抽取时尽力判断,判不准默认 person)
- **四个入口(手填 / DOI·URL / PDF / 题录)抽取出的作者,无论个人还是机构,一律原样写入 authorRaw**
  - 政府文件/法规:发布机构名(European Parliament、United States Congress、HKMA、MAS)→ `{name, type:"organization"}`
  - 题录无个人作者时(如"行业热搜"):用 `PB`(出版方)或 `JF`(期刊)兜底 → `type:"organization"`
- **硬必填判定**:`authorRaw` 非空即视为满足"≥1 作者"(机构名也算一个作者)
- 不阻塞入库;不为缺个人作者而拒收

> 关键:authorRaw 是过渡容器。它把"作者是谁"原样留住,等 Authors 模块建好再回填成正式实体。现在不要在上传管线里建 authors 表。

# 二、Authors 模块预定决策(排在上传管线之后那轮做,先登记)

> 现在不做,只登记,免得到那轮又重新讨论。依赖顺序:它和 Regulatory / Dashboard / Our Research 一批,在上传管线全部完成之后。

- `authors` 表:`id, name, type(person|organization), affiliation, researchInterests[], institutionId(可空)`
- `resource_authors` 多对多关联,带 `source(auto|manual)`(同标签的人工保护机制)
- **回填迁移**:遍历所有 resources 的 `authorRaw`,拆解成 authors 实体 + resource_authors 关联;同名合并去重(尤其跨资源共享作者,如 Ahmed/Aldasoro、Adrian)
- **机构作为可点击聚合实体**:点 "BIS" / "HKMA" / "European Parliament" → 看到它发布的全部资料(与标签"可点击聚合"同一逻辑);个人作者同理 → 作者页列出其在本平台的全部资源
- 回填完成后,authorRaw 可保留作审计或删除

# 三、sourceType 收口(最终枚举 + batch2 英文类型映射)

**最终枚举(中文八类,语言无关,英文资料也用这套):**
`期刊论文 / 工作论文 / 会议论文 / 学位论文 / 报告 / 政府文件 / 新闻 / 智库报告`

**batch2 夹具原来用的是英文旧类型,按下表映射校正**(校正版夹具见 `test-fixtures-batch2-urldoi-v2.json`):

| 英文旧值 | 中文枚举 | 适用样本 |
|---|---|---|
| Working Paper | 工作论文 | arXiv / NBER / Fed / SSRN(B2-01/02/03/12) |
| Gov Document | 政府文件 | MiCA / STABLE Act(B2-05/06) |
| Report(公司) | 报告 | Worldpay(B2-08) |
| Report(智库/机构) | 智库报告 | Coinbase Institute(B2-07) |
| News | 新闻 | St Louis Fed 博客(B2-09) |
| Journal Paper | 期刊论文 | Baur & Hoang / 可疑源(B2-10/11) |
| (会议论文集出版方) | 会议论文 | EWA Publishing 的 B2-04 |

> 题录导入的类型映射已在《题录导入第四入口》写过,与此一致(会议论文/学位论文/新闻等)。

5. Authors 模块那轮:建表 + 从 authorRaw 回填(本说明第二节)
