## Commit U.1 — 学者工具函数(`artifacts/api-server/src/lib/scholar/`)

每个函数独立、返回**标准化结构**(统一成同一个 `ScholarResult` 形状:title/authors/year/doi/canonicalUrl/fulltextUrl/accessStatus/venue/source),便于上层合并。

- `crossref.ts` → `searchCrossref(title)` / `resolveDoiCrossref(doi)`
  - 无需 key;请求头 `User-Agent: StablecoinHub/1.0 (mailto:<SCHOLAR_CONTACT_EMAIL>)` 进 polite pool
  - 端点:`https://api.crossref.org/works?query.bibliographic=<title>&rows=5`
- `openalex.ts` → `searchOpenAlex(title)`
  - 无需 key;所有请求带 `?mailto=<SCHOLAR_CONTACT_EMAIL>`
  - 返回作者 + 机构 ID(可喂将来的 authors/institutions)+ `best_oa_location`(OA 链接)
  - 端点:`https://api.openalex.org/works?search=<title>&per-page=5&mailto=<email>`
- `semanticscholar.ts` → `searchSemanticScholar(title)`
  - 无需 key(低速率);返回 `openAccessPdf`,对工作论文/预印本最强
  - 端点:`https://api.semanticscholar.org/graph/v1/paper/search?query=<title>&fields=title,authors,year,externalIds,openAccessPdf`
  - 做好 429 限流重试/退避
- `doi.ts` → `resolveDoi(doi)`
  - DOI content negotiation 验证存在并取元数据:
    `GET https://doi.org/<doi>` 带 `Accept: application/vnd.citationstyles.csl+json`
- `unpaywall.ts` → `unpaywall(doi)`
  - **必须**带 email:`https://api.unpaywall.org/v2/<doi>?email=<SCHOLAR_CONTACT_EMAIL>`
  - 有 DOI 但付费墙时找合法 OA 副本,填 `fulltextUrl` / `accessStatus`
- 验收:每个函数对已知样本(用测试夹具 #5 #6 #7)能独立返回正确标准化结果

## Commit U.2 — 反查链接 agent(`lib/scholar/resolveLink.ts`)

- 输入:抽取得到的 title / authors / year(可能缺 year)
- **先清洗标题**:去掉 PDF 图注/页码污染(如 `Image: Getty images...`、行尾 `1–12`)再检索 —— 见测试夹具 #8
- **多源瀑布**:Crossref → OpenAlex → Semantic Scholar,逐源尝试
- **匹配判定(防张冠李戴)**:候选记录需同时满足 ①标题词汇重合度高 ②作者姓氏有重合 ③年份接近(±1~2 年,year 缺失时此项放宽),三者交叉够吻合才接受
- **学术库全部未命中** → 不要直接 `not_found`,回退一次普通网页检索拿规范落地页(夹具 #8 的 WEF 文章就靠这步)
- 输出:区分 `canonicalUrl`(规范落地页/DOI)与 `fulltextUrl`(可打开的 OA 全文),另存 `accessStatus`(`open_access` / `publisher_paywalled` / `no_link_found`);并回填缺失的 year
- 都查不到 → 返回 `not_found`(交由 U.5 决定状态,不等于拒收)
- 验收:测试夹具 B 组(#5 #6 #7 命中学术库;#8 回退网页)全部按 expected 返回

## Commit U.4 — 核对 agent(`lib/verify.ts`)

入库前逐项验证,输出**逐字段核对报告** + 状态判定(不是简单通过/拒绝):
- **DOI**:`resolveDoi` 确认存在,且解析标题与卡片标题一致 → ✅;对不上 → ❌
- **URL**:可达/可解析 → ✅
- **作者/年份**:与权威记录(Crossref/OpenAlex)交叉 → 不一致标 ⚠️
- **摘要**:存在且非空 → ✅;缺 → ⚠️
- 报告结构:`{ field, status: '✅'|'⚠️'|'❌', detail }[]`
- 验收:夹具里 DOI 与标题对不上的情况该项标 ❌(可临时构造一条错配 DOI 测试)

## Commit U.5 — 状态机(对接已有 resourceStatusEnum)

- `resourceStatusEnum` 含:`uploading / processing / pending / needs_review / approved / rejected / failed`
- 判定规则:
  - 抽取失败(PDF 读不出文字且 OCR 也失败)→ `failed`
  - 核对全 ✅ → `pending`(普通用户)/ `approved`(管理员,沿用现有 isAdmin)
  - 有 ⚠️/❌ 或六要素不全 → `needs_review`(带核对报告进管理员队列)
- **完整性分两档**:硬必填(标题 + ≥1 作者 + 年份)缺才挡 `failed`;摘要 / 直达URL 缺照样入库标 `needs_review`。**不因缺一项整条丢弃**
- 验收:夹具 #8(无 DOI 但有 canonicalUrl + 摘要)→ 入 `needs_review`,不被拒

## Commit U.6 — 前端上传中心(`pages/academic-resources.tsx`)

三个 tab:手填 / DOI·URL / PDF。

### PDF 抽取:先抽文字,抽不到才 OCR(分两层)
- **第一层(本轮做)**:轻量库抽文字(`pdf-parse` 等,纯本地、快)。抽出的文字量超过阈值 → 直接喂 LLM 抽六要素
- **第二层(留接口,后续 commit)**:抽出文字几乎为空(扫描件/图片型)→ 触发 `ocrFallback(pdf)`。**本轮先把这个函数占位**(签名定好、内部先抛 "OCR 未启用"),架构上留好入口;真正接 Tesseract 排到电子版主线跑通之后
- 这样电子版全走快路,扫描件自动落到慢路,互不拖累

### 上传 UI
- 上传队列 + 进度条 + 百分比 + 当前任务 + 批量进度
- 三入口都支持:DOI·URL 支持单条 + 批量;PDF 支持单个 + 多个
- 核对报告在确认弹窗展示每字段 ✅/⚠️/❌,管理员可据此放行或退回
- 验收:三种方式可用;电子版 PDF 能抽出六要素;扫描件触发占位 OCR 给出明确提示;进度与核对结果可视
